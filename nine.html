Formatting tags :

  1. Strong
    • Use cases
      • Emphasizing Important Information
      Key Points in Articles or Blog Posts
      Highlighting Keywords or Search Terms
      Creating Stylish Headlines:
    • Best practice
      • Semantic Meaning: The "strong" tag should be used to convey semantic meaning rather than just for visual styling. It is intended to indicate content that is of particular importance or relevance within its context. Avoid using the "strong" tag solely for cosmetic purposes.
      Avoid Overuse: Use the "strong" tag sparingly and only on the most important or significant parts of your content. Overusing it can diminish its impact and make it less meaningful. Reserve it for truly important information that needs emphasis.
      Don't Rely on "strong" for Accessibility: While the "strong" tag may visually render as bold text by default, it is not meant to be solely relied upon for providing accessibility or making text stand out. Use appropriate CSS techniques to control the visual appearance of the text, ensuring it remains accessible to all users, including those with visual impairments.
      Consider Alternative Tags: Depending on the specific context, there may be alternative tags that better suit the meaning of your content. For example, if you want to highlight a specific keyword, the "mark" tag might be more appropriate, indicating a marked or highlighted section of the text.
  2. i
    • Use cases
      • Styling Book Titles or Foreign Words:
      Noting Variable Names or Technical Terms:
      Highlighting Quotes or Citations
      Author's Thoughts or Personal Opinions
    • Best practices
      • Consider Accessibility: Ensure that the use of the "i" tag doesn't hinder accessibility. Italics can be more challenging to read for some individuals, especially those with visual impairments. Avoid using italics excessively or for long blocks of text. If you have concerns about accessibility, consult accessibility guidelines and consider alternative ways to convey information without relying solely on visual cues.
      Use Descriptive Text: If you decide to use the "i" tag, ensure that the content within it is descriptive and meaningful on its own. Avoid using the "i" tag on text that doesn't provide any valuable information or context.
      Use for Presentational Purposes: The "i" tag should primarily be used for presentational purposes, such as applying italics to text, rather than conveying specific semantic meaning. Avoid using the "i" tag to add emphasis or importance to text. Instead, consider using appropriate semantic tags like "em" for emphasis or "strong" for stronger emphasis.
  3. u
    • Use cases
      • indicating Text Underline
    • Best practices
      • Use the "u" tag only for text that should be visually underlined.
      Avoid using the "u" tag solely for cosmetic purposes.
      Consider accessibility implications, as underlined text may be associated with hyperlinks.
  4. sup
    • Use cases
      • Representing Superscript Text
    • Best practices
      • Use the "sup" tag to indicate superscript text, typically used for mathematical exponents or footnotes.
      Avoid using the "sup" tag solely for cosmetic purposes.
      Consider accessibility implications, as superscript text may be associated with hyperlinks.
      Do not overuse the "sup" tag; use it only when necessary for clarity or proper notation.
  5. sub
    • Use cases
      • Representing Subscript Text
    • Best practices
      • Use the "sub" tag to indicate subscript text, typically used for mathematical subscripts or chemical formulas.
      Avoid using the "sub" tag solely for cosmetic purposes.
      Consider accessibility implications, as subscript text may be associated with hyperlinks.
      Do not overuse the "sub" tag; use it only when necessary for clarity or proper notation.
